Configuring the software

After the software has been installed, it must be configured. Follow the steps below for a successful configuration:

  1. Most configuration happens inside the meteo.xml(5) configuration file. A sample file is include with the distribution. To find out what sensor names are known for a particular station type, use the meteosensors(1) utility.
  2. A minimal installation will run one instance of meteopoll(1) to collect data from the weather station and write it to the database (with the -b mysql option to directly write the data to the database). It will also use meteoavg(1) to compute averages and store them in the averages table. Furthermore, it will run a cron job using meteodraw(1) to update the graphs for the web page, a sample cron job script is included in the distribution.
  3. More sophisticated installations will make use of the decoupling of weather station and database through a message queue, which involves running the meteopoll(1) program with the -b msgqueue option, and in addition running an instance of the dequeuer process meteodequeue(1).
  4. To top off the installation, configure the meteo browser to allow graphical browsing of the meteo database. See the README.browser for details.

For details of the configuration of all these programs, please consult the manual pages installed with the distribution.
